### 2-2. SEO 진단 항목별 분석

#### `<H1>` 요소가 2개 이상 발견 — 97개 페이지

**원인 확정**: `themes/[id]` 페이지마다 visible H1 + sr-only H1 공존

| 파일 | 라인 | H1 내용 | 가시성 |
|------|------|---------|--------|
| `themes/[id]/_components/detail-header/index.tsx` | L41 | `{theme.name}` | visible |
| `themes/[id]/page.tsx` | L279 | `{headline}` (sr-only 섹션) | hidden |

테마 목록 페이지도 동일 패턴:

| 파일 | 라인 | H1 내용 | 가시성 |
|------|------|---------|--------|
| `themes/_components/themes-header.tsx` | L88 | `테마 트래커` | visible |
| `themes/(list)/page.tsx` | L150 | `한국 주식시장 테마 생명주기 분석` | hidden |

**수정 방안**: sr-only 섹션의 `<h1>` → `<h2>`로 변경 (시맨틱 의미 유지, 크롤러 혼란 제거)

---

#### `<title>` 요소에 동일한 제목 — 110개 페이지

**원인 확정**: `generateMetadata` fallback이 고정 문자열

```typescript
// app/themes/[id]/page.tsx:83-84
if (!theme) {
  return { title: '테마 상세' }  // ← description 없음, 모든 실패 페이지 동일
}
```

비활성/삭제된 테마 ID가 네이버 색인에 잔존 → 전부 동일 제목 `테마 상세` 반환.

**수정 방안**:
- fallback에서 `notFound()` 호출 (404 반환) → 네이버가 색인에서 자동 제거
- 또는 `robots: { index: false }` 추가

---

#### `<meta name="description">` 태그에 동일 설명문 — 120개 페이지

**원인**: 위 fallback과 동일. theme null일 때 description 미제공 → 네이버가 자동 추출한 동일 텍스트를 설명문으로 표시.

**수정 방안**: 위와 동일 — `notFound()` 호출로 근본 해결

---

#### `<title>` 요소 찾을 수 없음 — 1개

**추정 원인**: 404 페이지(`not-found.tsx`)에 `metadata` export 없음. 글로벌 layout의 titleTemplate만 적용.

**수정 방안**: `not-found.tsx`에 metadata export 추가 또는 `app/not-found.tsx`를 layout 포함 구조로 변경

---

#### `<meta name="description">` 설명 누락 — 1개

**추정 원인**: 위와 동일 — 404 페이지 또는 theme fallback 케이스
